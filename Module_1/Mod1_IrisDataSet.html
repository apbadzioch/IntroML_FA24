Training and Testing¶

  • When assessing a model's performance, the labeled data (input data that has been annotated with the correct output or target value (each data point is paired with a label or category that it belongs to))

The data is broken into two parts, one to build the model

  • Training Data (Training Set) is usually about 70% - 80%

The rest is used to assess how well the model works

  • Test Data (Test Set) (Hold-out Set) is usually about 20% - 30%

K-Nearest Neighbors (k-NN)¶

  • The k in k-nearest neighbors allows any fixed number (k) of neighbors in the training (can be just about any number, but it is common to use odd numbers as even numbers can cause the algorithm to struggle when making decisions).

  • Using 1 can make the model sensitive to noise, leading to overfitting, but a very large k can cause the model to be too generalized, leading to underfitting.

  • Overfitting: occurs when a model learns not only the underlying patterns in the training data but also the noise and random fluctuations. This results in excellent performance on the training data but poor generalization to new, unseen data. Overfitting is like memorizing the training data rather than understanding the broader patterns.

  • Underfitting: happens when a model is too simple to capture the underlying patterns in the data. It fails to perform well even on the training data and consequently also performs poorly on new data. Underfitting occurs when the model doesn’t learn enough from the data, often because it’s too basic or constrained.
